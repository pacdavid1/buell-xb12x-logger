🏁 Fuel Trim — F7 + Sessions VS

ACTIVE Main tuning pipeline: compares two sessions, determines which map is better per cell
RAW INPUTS
📡 ECU Core
ecu/protocol.py → CSV_COLUMNS
RPM TPS_pct VS_KPH Gear pw1 pw2 CLT flags (fl_wot, fl_decel, fl_fc) baro_hPa
Alpha-N: TPS+RPM drive fueling. VS_KPH from VSS counts.
STAGE 1
📄 load_csv()
web/launch.py:load_csv()
← raw ECU signals
pw1_norm = pw1 × 1013.25/baro pw2_norm = pw2 × 1013.25/baro gear_detected = detect_gear() rows[] with normalized PW
Baro normalization contradicts CLAUDE.md rule "do NOT normalize PW by baro for DDFI2 Alpha-N"
STAGE 2
🏷️ classify()
web/launch.py:classify()
← rows[]
fl_wot OR tps≥85 → SPICY_WOT dtps>15 → SPICY_TIPIN flat TPS+RPM + flat alt → SWEET clt<170 OR wue>102 OR ae>105 → BITTER dalt>0.8 → SALTY_UP dalt<-0.8 → SALTY_DOWN flavor per row {SWEET, SPICY_*, SALTY_*, BITTER}
BITTER rows excluded from fueling analysis. SWEET = ideal tuning data.
STAGE 3
📊 build_index()
web/launch.py:build_index()
← classified rows[]
RPM→bin (800-6800, 15 bins) TPS→bin (0-100, 16 bins) pw_eff = (pw1_norm+pw2_norm)/2 × AFV/100 Welford online std_rpm, std_tps, std_pweff per-cell dict {n, pw1, pw2, pw_eff, std_*, ...}
AFV locked at 100.0 (O2 disconnected) — pw_eff = pw_norm × 1.0
STAGE 4
🎯 f7_event_detect()
web/f7.py:_f7_detect_events()
← rows[]
Stable bucket≥3s (rolling std RPM<150, TPS<2) TPS break ≥2% → event start Phase B PW curve captured tps_curve_norm for cross-session DTW Fuel cut trim (<2ms), PW drop trim (>35%) events[] with pw_curve, tps_curve, bucket_a, env context
Pure PW/TPS physics, no EGO dependency. BW: 800-6800RPM, 0-100%TPS
STAGE 5
🔗 f7_dtw_cluster → f7_cross_session_match
web/f7.py:_f7_cluster() / _f7_match_cross_session()
← events[] from both sessions
Complete-linkage DTW on PW curves (threshold 0.85) Sub-divide by Bucket A consistency DDTW on tps_curve_norm for cross-session match delta_pw, delta_vss, confidence, efficiency_delta matches[] with per-event PW delta
IDEA-033: DDTW for shape matching, not amplitude. Already implemented in _f7_ddtw().
STAGE 6
⚖️ _compare_sessions_cached()
web/vs_engine.py + web/launch.py:_compare_sessions()
← index a + index b + f7_matches
Common cells: SWEET + SPICY_WOT only dpw_eff = pw_eff_b − pw_eff_a GAP 1 Welch CI: dpw_eff ± 1.96×SE dpw_eff_sig = CI does not cross 0 F7+VS zone fusion: LIGHT=F7, MID=blend, WOT=VS delta[] with per-cell dpw_eff, CI, significance
Cache version 9. GAP1 computed but NOT yet consumed by merge (GAP backlogged).
UNVALIDATED
STAGE 7
🧩 _merge_maps() + GP Regression
web/vs_engine.py:_merge_maps() / _gpr_predict_grid()
← delta[] + f7_cells + eeprom_a + eeprom_b
GAP1 significance gate (skip if !dpw_eff_sig) Fusion: vs_delta ⊕ f7_delta weighted by confidence Bias toward richer (safer) when signs disagree GP Regression (Matern 5/2, scikit-learn) fills unfilled cells Winner: 'A' if dpw_eff<0, 'B' if >0 merged maps {fuel_front, fuel_rear, spark_front, spark_rear}
GP regression fills ~40% more cells than raw winner-take-all alone.
OUTPUTS
📤 Merged Maps → UI
← merged maps
ui_tuner: Δ maps with cell-level winners ui_sessions_vs: F7 matches overlay eeprom_bin: encoded for QUEMAR
Output feeds Map Editor (Sector 4) for burning to ECU.

🔥 VDYNO — Virtual Dyno

ACTIVE_UNVALIDATED Estimates wheel power from GPS acceleration + physics model
RAW INPUTS
📡 GPS + ECU + Environment
ride_csv columns
VS_KPH RPM TPS_pct Gear fl_wot fl_decel baro_hPa IAT_Corr gps_alt_m
STAGE 1
✂️ _extract_segments()
web/vdyno.py
Filter: fl_wot OR TPS≥70% Reject: fl_decel, gear=0, VS_KPH<20, RPM<1500 Min segment: 1.5s continuous WOT segments[]
STAGE 2
🔬 _seg_physics()
web/vdyno.py
← segments
VS_KPH → m/s → acceleration F = m×a + 0.5×ρ×CdA×v² + Crr×m×g P_w = F × v (watts) SAE J1349 correction from baro+IAT physics arrays {times, rpms, vss_s, P_w}
Config defaults: mass=295kg, CdA=0.60, Crr=0.015, ρ=1.10 (Mexico City)
UNVALIDATED
STAGE 3
📊 _build_result_bins()
web/vdyno.py
← physics arrays
RPM binning (250 RPM bins) Median kW per bin + p25/p75 HP = kW × 1.341, Torque = kW×1000/ω per-RPM-bin {hp, torque, n}
OUTPUTS
📤 Power Curves
ride_vdyno_json (per-ride cache) ride_vdyno_rows_json (per-row sparse) ui_sessions_vs (compare A vs B) ui_launch_power (launch clusters) compare_sessions() (delta per RPM bin)

⚠️ Cost of Captured Unused Signals

18 columns in CSV are NEVER consumed by any tuning decision.
At ~8Hz logging, that's 144 bytes/s × 3600s = ~500KB per ride of wasted storage and processing.
Biggest impact: Batt_V (BL-DI-01) — empirically confirmed to cause ~2.3% pw shift that masquerades as map differences in Sessions VS.
Easiest cleanup: rider_notes (dead field), O2_ADC (dead signal), humidity_pct (descriptive only).
