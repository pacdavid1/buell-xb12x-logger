02 / 11

Hardware Stack

What runs where

🔴 Pi Zero 2W

  • Runs Python 3 multiprocess
  • Main process + ECU subprocess
  • IPC via tmpfs files
  • Web server on :8080
  • Wi-Fi AP + client modes
  • CH343P USB-Serial adapter

⚙️ ECU DDFI2

  • Delphi DDFI2 — Alpha-N
  • Open Loop (no O2 sensor)
  • Serial 9600 8N1 protocol
  • 80+ parameters per frame
  • VE + Spark maps (12x11)
  • EEPROM read/write via K-Line

📡 Sensors

  • BMP280 — barometric pressure
  • AHT20 — intake air temp
  • CW2015 — battery monitor
  • Ublox GPS — position + speed
  • All via I²C bus
  • Logged at 1–8 Hz
Pi ⇄ CH343P ⇄ DDFI2 ⇄ Buell XB12X (1203cc V-Twin)

VDYNO Program

Virtual Dyno — Roadmap V0 → V4

V0 · Burn Ledger

Complete audit trail of all EEPROM burns. Each write logged with timestamp, session hash, cell deltas, and rollback capability. ✓ Done

V1 · Physical Power

Estimate rear-wheel HP from logged VSS + acceleration. Use known drag coefficients (CdA, rolling resistance) for the XB12X. Compare before/after burns.

V2 · Session Pairs

Auto-pair matching sessions by GPS route + weather. Normalize for wind, gradient, temperature. Deliver "this tune vs that tune" HP delta with confidence.

V3-V4 · AI Directed

Targeted data gathering: AI identifies VE cells with high uncertainty and suggests specific RPM/TPS regions to log. Fully automated optimization loops.
